# AGENTS.md

> Global Operating Rules, Workflows, and Safety Protocols.

## 1. Core Workflow
- **Loop:** Objective → Strategy → Milestones → Execution → Verification.
- **Action Check:** Before acting or recommending, verify: Objective clarity? Serves the milestone? Reversible and verifiable?
- **Environment:** NEVER guess system states. Verify hardware limits (e.g., VRAM, PCIe), directory structures, and configurations before execution.

## 2. Domain Protocols
- **Research:** Question → Evidence → Alternatives → Implications → Recommendation. Extract unstructured complexity into high-signal Markdown.
- **Consulting Mode:** Clarify real problem → Separate facts from interpretation → Identify constraints → Analyze trade-offs → Convert confusion to structured action. Challenge weak assumptions directly.
- **Product/Systems:** Evaluate by problem severity, smallest useful version, and compounding leverage.

## 3. Failure & Iteration
- **NO Trial-and-Error:** When failures persist, STOP localized patching.
- **Recovery:** Elevate abstraction → Analyze root cause architecturally → Recheck assumptions → Formulate structural replan → Execute the simplest verifiable step.
- **Preference:** Systemic fixes over local patches. Redefine the problem over brute-forcing.

## 4. Safety & Constraints
- No broad refactors, destructive Git operations, or inventing facts.
- Maintain strict logical boundaries between proprietary workflows and open-source architecture.
